1. JAVA의 역사

    JAVA는 1991년 썬 마이크로시스템즈사에서 제임스 고슬링이 고안하였고 오크 호은 그린이라 불렸다. 오크는 최초 가전제품에서 쓰일 프로그램의 개발이 목적이였지만 인터넷과 웹의 출현으로 자바의 초점은 가전제품에서 인터넷으로 옮겨지게 되었고 1995년 이름을 자바로 변경하게 되었다.
    고슬링의 목표는 C/C++ 스타일의 언어와 가상머신을 구현하는 것으로 Write Once,  Run Anywhere 하는 것을 약속하엿다.

2. 특징

    1) JAVA는 객체지향 언어이다.

    여기서 객체지향 프로그램이란 프로그램을 개발하는 기번으로 부품에 해당하는 객체들을 먼저 만들고 이것들을 하나씩 조립 및 연결하여 전체 프로그램을 완성하는 것을 의미한다.

    객체를 만들기 위해서 는 설계도인 클래스를 작성하고 객체와 객체를 연결하여 목적에 맞는 프로그램을 만들어 낸다.

    객체지향 언어의 특징인 캡슐화 상속성 다형성을 완벽하게 지원할 수 있다.


    2) 이식성이 높다.

    서로 다른 실행환경을 가진 시스템간에 프로그램을 옴겨 실행할 수 있는 것을 말한다.

    자바는  자바 실행 환결(JRE)이 설치 되어 있는 모든 운영체제 에서 실행이 가능하다.

    3) 인터프린터 언어이다.
     
    컴파일 언어인 동시에 인터프리터 언어이다.

    텍스트 소스를 컴파일 하여 2진 파일로 만든 다음 자바 런타임 클래스 파일을 인터프리트 하면서 실행

    시스템 무관한 2진 파일을 만듬으로써 자바는 컴파일 언어에 가까운 속도와 시스템 독립성을 얻을 수 있다.

    4) 메모리를 자동을 관리한다

    C++은 메모리에 생서된 객체를 제거하기 위해 개발자가 직접 코드를 작성해야한다. 만약 이 작업을 성실하게 해주지 않는다면 프로그램은 불완전해지고 갑자기 다운되는 현상을 겪을수도 있다.

    자바는 개발자가 직접 메모리에 접근할 수 없으며 메모리는 자바가 직접 관리한다.

    객체 생성시 자동으로 메모리 영역을 찾아서 할당하고 사용이 완ㄹ료되면 쓰레기 수집기를 실행시켜 자동울 사용하지 않는 객체를 제거한다.
